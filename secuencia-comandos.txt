📘 INSTRUCCIONES PARA EJECUCIÓN DE SCRIPTS EN POSTGRESQL
────────────────────────────────────────────────────────────

Este documento describe dos métodos para ejecutar scripts SQL en PostgreSQL:
1. Desde la terminal de Windows (CMD o PowerShell)
2. Desde la consola interactiva de `psql`

📌 Requisitos previos:
- Tener PostgreSQL instalado y configurado correctamente.
- Contar con los archivos `dbCreate.sql`, `loadData.sql` y `runStatements.sql` en una carpeta accesible.
- Conocer el nombre de usuario de PostgreSQL (por defecto suele ser `postgres`).

────────────────────────────────────────────────────────────
🖥️ MÉTODO 1: EJECUCIÓN DESDE LA TERMINAL DE WINDOWS
────────────────────────────────────────────────────────────

🔹 Paso 1: Abrir la terminal
- Presione `Win + R`, escriba `cmd` y presione Enter.
- Alternativamente, puede buscar "PowerShell" en el menú de inicio y abrirlo.

🔹 Paso 2: Navegar al directorio donde están los archivos `.sql`
cd C:\ruta\completa\al\proyecto

🔹 Paso 3: Ejecutar los comandos en orden

REM 1. Crear la base de datos
psql -U postgres -h localhost -c "CREATE DATABASE estacionamiento;"

REM 2. Crear las tablas
psql -U postgres -h localhost -d estacionamiento -f dbCreate.sql

REM 3. Cargar los datos
psql -U postgres -h localhost -d estacionamiento -f loadData.sql

REM 4. Ejecutar las consultas (solo si la carga fue exitosa)
psql -U postgres -h localhost -d estacionamiento -f runStatements.sql

────────────────────────────────────────────────────────────
🧑‍💻 MÉTODO 2: EJECUCIÓN DESDE LA CONSOLA INTERACTIVA DE PSQL
────────────────────────────────────────────────────────────

🔹 Paso 1: Abrir la consola `psql`
- En la terminal de Windows, escriba:
psql -U postgres

🔹 Paso 2: Navegar al directorio donde están los archivos `.sql`
\cd /ruta/completa/al/proyecto

🔹 Paso 3: Ejecutar los comandos en orden

-- 1. Crear la base de datos
CREATE DATABASE estacionamiento;

-- 2. Conectarse a la base de datos recién creada
\c estacionamiento

-- 3. Crear las tablas
\i dbCreate.sql

-- 4. Cargar los datos
\i loadData.sql

-- 5. Ejecutar las consultas (solo si no se presentaron errores en la carga)
\i runStatements.sql
